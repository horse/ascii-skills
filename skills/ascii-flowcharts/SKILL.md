# ASCII 流程图技巧

## 核心理念

流程图展示**步骤顺序**和**逻辑分支**，让流程可视化。

## 1. 基本元素

### 节点类型

```
开始/结束:     [开始]    (椭圆/圆角)
               [结束]    

处理步骤:      ┌─────┐
               │处理  │  (矩形)
               └─────┘

判断/分支:      ┌─────┐
               │判断? │  (菱形)
               └─────┘

输入/输出:      ──────►
               输入    (平行四边形模拟)
```

### 连接箭头

```
向右:    ──────►
向左:    ◄──────
向上:      ▲
          │
          ▼
向下:      ▼
          │

双向:    ◄────►

折线:
    ────►
          │
          ▼
          │
    ◄──────
```

## 2. 线性流程

### 简单步骤链

```
[开始] ──► [步骤1] ──► [步骤2] ──► [步骤3] ──► [结束]
```

### 带说明的步骤

```
┌──────────────┐
│  1. 初始化   │ ──►
└──────────────┘
      │
      ▼
┌──────────────┐
│  2. 处理数据 │ ──►
└──────────────┘
      │
      ▼
┌──────────────┐
│  3. 输出结果 │
└──────────────┘
```

### 编号步骤

```
① 读取输入
   │
   ▼
② 验证数据
   │
   ▼
③ 处理业务
   │
   ▼
④ 返回结果
```

## 3. 分支结构

### 条件分支（Yes/No）

```
              ┌─────────┐
              │ 判断条件 │
              └────┬────┘
                   │
         ┌─────────┴─────────┐
         │                   │
      Yes │                │ No
         ▼                   ▼
   ┌─────────┐        ┌─────────┐
   │  分支 A │        │  分支 B │
   └────┬────┘        └────┬────┘
        │                   │
        └─────────┬─────────┘
                  ▼
            [合并后继续]
```

### 多分支

```
┌──────────────┐
│  判断类型    │
└──────┬───────┘
       │
  ┌────┼────┬────┐
  ▼    ▼    ▼    ▼
 类型1 类型2 类型3 其他
  ▼    ▼    ▼    ▼
[执行相应处理]
```

### 完整示例

```
                          ┌───────────────┐
                          │  用户输入    │
                          └───────┬───────┘
                                  │
                                  ▼
                         ┌────────────────┐
                         │  验证输入     │
                         └───────┬────────┘
                                 │
                  ┌──────────────┼──────────────┐
                  ▼                                 ▼
         ┌────────────────┐              ┌────────────────┐
         │   输入无效     │              │   输入有效    │
         │   显示错误    │              └───────┬────────┘
         └───────┬────────┘                      │
                 │                                ▼
                 │                    ┌────────────────────┐
                 └───────────────────►│   处理业务逻辑    │
                                       └─────────┬──────────┘
                                                 │
                                                 ▼
                                       ┌────────────────────┐
                                       │    返回结果        │
                                       └────────────────────┘
```

## 4. 循环结构

### while 循环

```
┌───────────────┐
│   开始循环   │
└──────┬───────┘
       │
       ▼
  ┌─────────┐
  │ 条件判断│──────否──► 退出
  └────┬────┘
       │ 是
       ▼
  ┌─────────┐
  │ 执行循环│
  │  体内容 │
  └────┬────┘
       │
       └───────┘
         │
         ▼
    (返回条件判断)
```

### for 循环

```
┌───────────────┐
│  i = 0       │
└──────┬───────┘
       │
       ▼
  ┌─────────┐
  │ i < n?  │────否──► 结束
  └────┬────┘
       │ 是
       ▼
  ┌─────────┐
  │  处理   │
  │ i++     │
  └────┬────┘
       │
       └───────┘
         │
         ▼
    (返回条件判断)
```

## 5. 泳道图（多角色流程）

```
┌─────────────────────────────────────────────────────────────┐
│                      用户流程                              │
├─────────────────┬─────────────────┬─────────────────────────┤
│    用户         │    系统         │    管理员               │
├─────────────────┼─────────────────┼─────────────────────────┤
│                 │                 │                         │
│  [提交请求]     │                 │                         │
│      │         │                 │                         │
│      ▼         │                 │                         │
│      ├─────────►│                 │                         │
│                 │  [验证请求]     │                         │
│                 │      │         │                         │
│                 │      ▼         │                         │
│                 │      ├─────────►│                         │
│                 │                 │   [审核]                │
│                 │                 │      │                 │
│      ◄──────────│                 │      ▼                 │
│  [收到结果]     │                 │      │                 │
│                 │                 │                         │
└─────────────────┴─────────────────┴─────────────────────────┘
```

## 6. 实用模板

### 决策流程

```
                          ┌─────────────┐
                          │   开始      │
                          └──────┬──────┘
                                 │
                                 ▼
                         ┌─────────────┐
                         │  有权限？   │
                         └──────┬──────┘
                    ┌──────────┴──────────┐
                    ▼                         ▼
            ┌─────────────┐          ┌─────────────┐
            │   允许访问  │          │   拒绝访问  │
            └──────┬──────┘          └──────┬──────┘
                   │                         │
                   └───────────┬─────────────┘
                               ▼
                       ┌─────────────┐
                       │    结束     │
                       └─────────────┘
```

### 错误处理流程

```
┌─────────────┐
│   执行操作   │
└──────┬──────┘
       │
       ▼
  ┌─────────┐
  │ 成功？  │
  └────┬────┘
   是  │  否
   ▼   ▼
┌──────┐ ┌─────────────┐
│ 继续 │ │ 错误处理   │
└──────┘ └──────┬──────┘
       │        │
       │        ▼
       │  ┌─────────────┐
       │  │ 记录日志   │
       │  └──────┬──────┘
       │         │
       │         ▼
       │  ┌─────────────┐
       └──►│  返回错误  │
            └─────────────┘
```

### API 调用流程

```
  [客户端]                    [服务器]
     │                           │
     │    ┌─ Request ──────┐     │
     ├────►  GET /api/data  │     │
     │    └────────────────┘     │
     │                           │
     │                    ┌──────┴──────┐
     │                    │  验证请求   │
     │                    └──────┬──────┘
     │                           │
     │    ┌─ Response ──────┐    │
     ◄────── 200 OK + JSON  │    │
     │    └────────────────┘    │
     │                           │
     ▼                           ▼
```

## 7. 快速参考

```
常用模式:
线性:    A ──► B ──► C ──► D

分支:    ┌──► B
         │
    A ───┤──► C
         │
         └──► D

循环:    ┌──────────┐
         ▼         │
      ┌────┐       │
      │判断│──否──►[结束]
      └──┬─┘       │
       是│         │
         ▼         │
       [处理]──────┘

泳道:    │角色1│角色2│角色3│
         │  ───┼─────┼──►  │
```
