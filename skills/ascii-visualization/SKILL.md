# ASCII 综合可视化

## 核心理念

综合运用多种 ASCII 技巧，创建复杂、专业的信息可视化。

## 1. 工具组合策略

```
┌─────────────────────────────────────────────────────────┐
│                    选择合适的工具                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   数据类信息 ──────────────► 表格 (ascii-tables)        │
│   步骤/流程 ──────────────► 流程图 (ascii-flowcharts)  │
│   层级/树状 ──────────────► 树形图 (ascii-tree-diagrams)│
│   区域/组织 ──────────────► 方框布局 (ascii-box-layouts)│
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 3. 布局原则

### 信息密度平衡

```
太高密度:                          适当密度:
┌────┬────┐                        ┌────────┬────────┐
│A │ B │ C │                        │   A    │   B    │
├────┼────┤                        ├────────┼────────┤
│D │ E │ F │                        │   C    │   D    │
└────┴────┘                        └────────┴────────┘
  拥挤                                舒适
```

### 视觉层次

```
第一层 (标题):
═══════════════════════════

第二层 (模块标题):
┌──────────────────────────┐

第三层 (内容):
│ 内容内容内容

第四层 (细节):
  • 列表项
  • 列表项
```

### 一致性原则

```
✓ 统一边框风格
✓ 统一间距
✓ 统一对齐方式
✓ 统一缩进级别

✗ 混用边框
✗ 忽大忽小
✗ 位置不一
```

```
       ┌────┐
       │    │
       └────┘
       
步骤4: 填充内容
       ┌─────────┐
       │  服务器 │
       └────┬────┘
            │
            
步骤5: 调整优化
       对齐、间距、美化
```
